I might make an entirely separate detailed step by step post later on how to use databinding.

-Data binding
You should know how to get a reference to a specific view using the findViewById method. Remember, when you call findViewById method. The android system has to search through the hierarchy to find it. Typically in small apps it isn’t really a problem that would cause the app to slow down. However, in very large apps it could definitely have a large impact on performance. This is why data binding is very important. Data binding is creating an object that contains a reference to views in the xml. Data bindings main purposes are showing data, handling user events, and calling actions on layout variables. Some of the benefits from data binding are: less code, faster apps since the binding isn’t done in onCreate, fewer errors, and easier to maintain due to less code. -LinearLayout
A linear layout is a view group that arranges its child views horizontally(Left to right) or vertically(Up and down).

-ConstraintLayout
A constraint layout is a view group that lets you position and size the views in an extremely flexible way. The views are positioned using constraints. If you don’t have a constraint, it will shift when you load the app. The view needs at least one horizontal and vertical constraint. Constraint layouts are great because it offers a ton of flexibility when it comes to designing for screens with different sizes. Constraint layouts usually have a flatter view hierarchy than linear layouts. By flat view hierarchy, I mean no nested view groups. Constraint layouts allow for chaining, which is a pretty neat feature. It sounds exactly like it sounds. It allows you to connect a group of views together.

-Baseline Constraints
A baseline constraint aligns a textview’s text to the baseline of a different views text. This is very useful when views have different font sizes.

-ScrollView
A scroll view is great when you need to display a very large amount of content on a screen. It allows you to scroll from the very top of the content to the bottom. It’s very common to put a LinearLayout inside of a ScrollView. A very important note is ScrollView can only have one child. I should say this again, they can only have ONE child.

-ViewGroup
A ViewGroup is a view that contains other views inside of it. So you can imagine a ConstraintLayout with a LinearLayout inside.

-Styles
A style is a collection of attributes such as font color, font size, background color, padding, etc.. You can think of them as things you can use to change the look of a view. They also allow you to consistently style views when created.

-Click listeners
You use them to make a view respond to a certain action such as a click by the user. The function that makes the click listener receives the view that is clicked. There are two ways to attach a click listener to a view.

In the xml layout, you add the android:onClick attribute to the view element.
You can do it programmatically using the setOnClickListener(View.OnClickListener) function in the activity that the function interacts with.
