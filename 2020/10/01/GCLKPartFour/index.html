Google CodeLabs Kotlin Part Four

Posted on 2020-10-01 In Google , Kotlin , CodeLabs , Activity lifecycle , Fragment lifecycle , Lifecycle libraries , Lifecycle tips , Bundles Facebook: 0

Lesson 4 in the google code labs was all about the life cycles of activities and fragments. It also included information about using Timber for logging, life cycle tips, life cycle libraries, saving activity state with bundles, and configuration changes.

-Activity life cycle
There are seven life cycle methods for an activity. To add behavior to these life cycles, just override the method.
1.onCreate() - Called only once during lifetime of an activity instance
2.onStart()
3.onPause()
4.onRestart()
5.onResume()
6.onStop()
7.onDestroy() - Called only once during lifetime of an activity instance

-Fragment life cycle
1.onAttach() - Called when the fragment is associated with its activity
2.onCreate() - Fragment is created
3.onCreateView - Inflates the fragments layout
4.onViewCreated() - Called once onCreateView has returned
5.onStart() - Called when fragment becomes visible
6.onResume() - Called when fragment gains focus
7.onPause() - Called when the fragment loses focus
8.onStop() - Fragment is no longer visible
9.onDestroyView() - Fragment is no longer needed
10.onDestroy()
11.onDetach()

-Logging with android logging api
The log class allows you to write short messages that can be displayed in the logcat in android studio. To do so, you use Log.I(name of the class, message)

-Logging with Timber
Timber is a logging library that has some advantages over android’s logging api. Some of the benefits are that it generates the log tag for you based on the class name. Avoids showing logs in a release version of your app. Lastly it allows for integration with crash reporting libraries.

To use Timber:
1.Add its dependency to your Gradle file and extend the Application class to initialize it. Application is a base class that contains global application state for your entire app. There is a default Application class that is used by Android if you don’t specify one. You can create your own Application subclass to initialize app-wide libraries such as Timber.

2.Add your custom Application class to your app by adding the android:name attribute to the element in the Android manifest. Make sure to do this otherwise it will not work.
Use Timber.i() to write log messages with Timber. Remember the method only takes one argument, which is the message.

-Lifecycle Tips
If you set something up in a lifecycle callback, make sure to stop it in a corresponding callback. For example if you set something up in onCreate, make sure to stop it in onStop. Use onCreate to initialize parts of your app that will only run once. Be cautious of where you initialize parts of your app in the lifecycle callbacks.

-Lifecycle library
The android lifecycle library allows you to change the control from an activity or fragment to the component that needs to be aware of the life cycle. Life Cycle owners have/own life cycles such as Activity and Fragment. Life Cycle owners implement the lifecycle interface. Lifecycle observers pay attention to the current lifecycle state and perform tasks when the lifecycle changes. Lifecycle observers implement the LifecycleObserver interface. Lifecycle objects contain the actual lifecycle states, and they trigger events when the lifecycle changes.

To create a lifecycle aware class:
1.Implement the lifecycle observer interface in classes that need to be aware of the life cycle.
2.Initialize the observer with the object in the class or fragment
3.In the life cycle observer class, annotate lifecycle aware methods with the lifecycle state change they are interested in. For example, the @OnLifecycleEvent(Lifecycle.Event.ON_START)annotation indicates that the method is watching the onStart lifecycle event.

-Bundles
Bundles are a great way to save and store data for when the user exits your app. A bundle is a key value pair. The keys are always strings.

To use bundles:
1.Use the onSaveInstanceState() callback to save other data to the bundle that you want to retain, even if the app was automatically shut down. To put data into the bundle, use the bundle methods that start with put, such as putInt(). You can get data back out of the bundle in the onRestoreInstanceState() method, or more commonly in onCreate(). The onCreate() method has a savedInstanceState parameter that holds the bundle.If the savedInstanceState variable contains null, the activity was started without a state bundle and there is no state data to retrieve.

2.To retrieve data from the bundle with a key, use the Bundle methods that start with get, such as getInt().
